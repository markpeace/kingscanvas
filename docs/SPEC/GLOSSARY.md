# Glossary

Canonical terms to ensure consistent prompts and docs.

- **Epoch**: A bounded chunk of work in the roadmap.
- **Prompt**: A structured instruction received by the back-end AI.
- **PR (Pull Request)**: GitHub unit of change; must be logged.
- **STATE**: Single source of truth; CURRENT.yaml points to active work.
- **HISTORY**: Immutable archive of completed epochs.
- **Export**: Curated set of files for front-end AI.
- **Invariants**: Rules that must always hold true (e.g., UTC IDs).
- **CI**: Continuous Integration; validates docs, schemas, and consistency.
